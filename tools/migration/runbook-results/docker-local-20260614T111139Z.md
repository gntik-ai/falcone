## Notes

- This dry-run exercised the **data-plane** migration end-to-end (gate ‚Üí snapshot ‚Üí
  sync ‚Üí snapshot ‚Üí compare ‚Üí idempotency) against real MinIO + real SeaweedFS. The
  Helm re-point (Step 4) and external traffic switch (Step 6) are control-plane/cluster
  operations validated on the kind test cluster; the chart re-point command is
  template-verified here.
- ETag parity holds for single-part objects across backends. For very large
  (multipart) objects, ETags can differ between engines; the downstream
  `add-seaweedfs-migration-validation` change is responsible for content-hash parity on
  multipart objects beyond ETag equality.
