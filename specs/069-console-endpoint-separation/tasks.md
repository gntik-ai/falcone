### T03-P2-02 — Enforce console separation validation rules

**File**: `scripts/lib/public-api.mjs`

**Action**:
Add validation rules to `collectOperationViolations()` / `collectRouteCatalogViolations()`:

1. If an operation declares any `x-console-*` metadata, it must declare `x-console-tier` with one of `spa`, `backend`, `platform`.
2. If `x-console-delegation-mode === "backend_workflow"`, the operation must also declare `x-console-workflow-id` matching `^WF-CON-0[0-9]{2}$`.
3. If `x-console-status-operation-ids` is present, every referenced operation ID must exist in the route catalog after regeneration.
4. The route catalog must preserve the console fields exactly from the source OpenAPI.
5. `getTenantWorkflowJobStatus` must stay tenant-scoped and `getWorkspaceWorkflowJobStatus` must stay workspace-scoped.

**Note on deny-by-default**:
In this repo, deny-by-default is represented as **validation failure for any console-governed route missing explicit tier metadata**. Do not attempt to redesign APISIX family routing in T03.

**Done criterion**:
`npm run validate:public-api` fails when console-governed routes are unlabeled/misaligned and passes once the metadata is correct.
