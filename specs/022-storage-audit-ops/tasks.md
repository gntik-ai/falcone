## Phase 2 — Internal helpers

- [ ] T006 Implement internal (non-exported) helper `sanitizeAuditString(value)` in `storage-audit-ops.mjs`. Accepts any value. If `value` is not a string, returns `value` unchanged. Strips URLs (any substring matching `https?://[^\s]+`) replacing them with `[redacted-url]`. Strips secret key material patterns (base64-like sequences of 20+ characters matching `[A-Za-z0-9+/]{20,}={0,2}`) replacing them with `[redacted]`. Mirrors the `sanitizeAuditString` convention from `storage-event-notifications.mjs`. This helper is called on every string field before an audit event is frozen.
- [ ] T007 Implement internal helper `generateEventId(seed)` in `storage-audit-ops.mjs`. Accepts an optional `seed` string. Uses `node:crypto` (`createHash('sha256')`) on `(seed ?? '') + Date.now()`. Returns the first 32 characters of the hex digest prefixed with `sevt_` (total length: 37 characters). When `seed` is omitted, the result is random per call; when provided, the result is deterministic for the same seed+timestamp combination.
- [ ] T008 Implement internal helper `assertAuditQueryScope(context, params)` in `storage-audit-ops.mjs`. Enforces FR-019, FR-020, FR-021. Accepts `context` (with fields: `actorId`, `actorType`, `tenantId`, `workspaceId`, `isSuperadmin`) and `params` (the raw query filter object). Rules: (a) if `context.isSuperadmin` is true, no restriction applies; (b) if `actorType` is `'tenant_owner'`, `params.tenantId` must equal `context.tenantId` — else throw `AUDIT_SCOPE_UNAUTHORIZED`; (c) if `actorType` is `'workspace_admin'`, `params.tenantId` must equal `context.tenantId` AND `params.workspaceId` (when present) must equal `context.workspaceId` — else throw `AUDIT_SCOPE_UNAUTHORIZED`; (d) if `actorType` is `'developer'` or `'user'`, `params.tenantId` must equal `context.tenantId`, `params.workspaceId` must equal `context.workspaceId`, AND (`params.actorId` must equal `context.actorId` OR `params.credentialId` must be a credential owned by `context.actorId`) — else throw `AUDIT_SCOPE_UNAUTHORIZED`. Thrown errors must carry `{ code: 'AUDIT_SCOPE_UNAUTHORIZED' }` in their `detail` property.
- [ ] T009 Implement internal helper `normalizeQueryParams(params)` in `storage-audit-ops.mjs`. Validates and normalizes query filter params before passing to the audit store. Enforces: `limit` is coerced to integer, clamped to `[1, 500]` (default 50) — throws `AUDIT_QUERY_INVALID` when the provided value is not a positive integer; `fromTimestamp` and `toTimestamp` when present must be valid ISO-8601 strings (parseable by `Date.parse` without `NaN`) — throws `AUDIT_QUERY_INVALID` otherwise; `sortOrder` defaults to `'desc'` when not provided; unknown filter fields are passed through without modification. Returns a new normalized params object.
- [ ] T010 Implement internal constant `CATEGORY_OPERATION_TYPE_MAP` in `storage-audit-ops.mjs` (non-exported). A frozen Map (or frozen plain object) that maps each of the 15 `STORAGE_AUDIT_COVERAGE_CATEGORIES` string values to an array of `STORAGE_AUDIT_OPERATION_TYPES` value strings that belong to that category. Example: `'bucket.create'` → `['bucket.create']`; `'credential.lifecycle'` → `['credential.create', 'credential.rotate', 'credential.revoke', 'credential.expire']`; `'access.denied'` → `['access.denied']`; `'object.write'` → `['object.put']`. Every coverage category must map to at least one operation type.

## Phase 3 — Unified schema builder

- [ ] T011 Implement and export `buildStorageUnifiedAuditEvent(input)` in `storage-audit-ops.mjs`. This is the low-level builder that all higher-level builders delegate to. Accepts an input object with all fields from the unified schema (FR-001, FR-002). Mandatory fields that must be non-null in the output: `eventType`, `operationCategory`, `operationType`, `actorId`, `actorType`, `outcome`, `occurredAt`. Optional fields default to `null` when absent: `credentialId`, `objectKey`, `changeSummary`, `errorCode`, `policySource`, `triggerSource`, `cascadeTriggered`, `cascadeScope`, `correlationId`. Applies `sanitizeAuditString` to every string field in the output object. Assigns `eventId` from `generateEventId(input.eventId ?? input.correlationId)` when not supplied. Sets `occurredAt` to `new Date().toISOString()` when not supplied. Sets `resourceScope` to `{ tenantId: null, workspaceId: null, bucketId: null, objectKey: null }` merged with provided `resourceScope`. Returns a deeply-frozen plain object (Object.freeze on the top-level record and on `resourceScope`).

## Phase 4 — Administrative audit builders

- [ ] T012 Implement and export `buildStorageAdminAuditEvent(operationType, context, detail)` in `storage-audit-ops.mjs`. Covers all 14 administrative operation types defined by FR-007. Parameters: `operationType` (one of the administrative `STORAGE_AUDIT_OPERATION_TYPES` values — throws a structured guard error if an unsupported type is passed); `context` (`{ actorId, actorType, tenantId, workspaceId, correlationId }`); `detail` (`{ bucketId?, outcome, changeSummary?, cascadeTriggered?, cascadeScope?, triggerSource? }`). Delegates to `buildStorageUnifiedAuditEvent` with `operationCategory: 'administrative'`, `eventType: 'storage.' + operationType`. Returns the frozen unified event record.
- [ ] T013 Enforce `changeSummary` shape for policy-change operations inside `buildStorageAdminAuditEvent`. When `operationType` is one of `bucket_policy.create`, `bucket_policy.update`, `bucket_policy.delete`, `bucket_policy.superadmin_override`, `workspace_permissions.update`, `tenant_template.update`: if `detail.changeSummary` is provided, it MUST be an object containing only the keys `statementsAdded`, `statementsRemoved`, `statementsModified` (integers ≥ 0) and optionally `statementId` (string). Any other keys in `changeSummary` must be silently stripped (not thrown) before the event is built. This prevents full policy documents from appearing in audit events (FR-008).
- [ ] T014 Enforce `changeSummary` shape for quota-change operations inside `buildStorageAdminAuditEvent`. When `operationType` is one of `quota.tenant_update`, `quota.workspace_update`: if `detail.changeSummary` is provided, it MUST be an object containing only `{ dimension, oldValue, newValue }` fields. Strip any other keys silently. Old and new values are numeric (quota limits) and are not secrets — they may appear in the audit event as-is.
- [ ] T015 Enforce `cascadeTriggered`/`cascadeScope` population for lifecycle operations inside `buildStorageAdminAuditEvent`. When `operationType` is one of `tenant_context.suspend`, `tenant_context.delete`: if `detail.cascadeTriggered` is not explicitly `false`, it MUST be a boolean and `detail.cascadeScope` MUST be a plain object (e.g., `{ credentialsRevoked: number, workspacesAffected: number }`). Throw a structured guard error if `cascadeTriggered: true` but `cascadeScope` is null/undefined. Per FR-009.

## Phase 7 — Event normalization

- [ ] T020 Implement and export `normalizeStorageAuditEvent(sourceEvent)` in `storage-audit-ops.mjs`. Pure function. Accepts any existing storage audit event record produced by the five heterogeneous builders: `buildStorageMutationEvent`, `buildStorageErrorAuditEvent`, `buildStorageEventNotificationAuditEvent`, `buildStorageUsageAuditEvent`, `buildStorageImportExportAuditEvent`. Determines `operationCategory` by inspecting `sourceEvent.eventType` using `categoryForEventType(eventType)` helper (T010): mutation events (`storage.object.*`, `storage.bucket.*`) → `'data_plane'`; error events (`storage.error.*`) → `'error'`; event-notification audit (`storage.event_notification.*`) → `'lifecycle'`; usage audit (`storage.usage.*`) → `'administrative'`; import/export audit (`storage.import_export.*`) → `'data_plane'`; unknown → `'data_plane'`. Per FR-006.
- [ ] T021 Map source event fields to unified schema fields inside `normalizeStorageAuditEvent`. Field mapping: `sourceEvent.actorUserId` or `sourceEvent.actingPrincipal?.id` → `actorId`; `sourceEvent.actingPrincipal?.type` → `actorType` (default `'user'`); `sourceEvent.credentialId` → `credentialId` (null when absent); `sourceEvent.correlationId` → `correlationId`; `sourceEvent.occurredAt` or `sourceEvent.timestamp` → `occurredAt`; `sourceEvent.outcome` → `outcome`; `sourceEvent.bucketId` or `sourceEvent.resourceScope?.bucketId` → `resourceScope.bucketId`; `sourceEvent.workspaceId` or `sourceEvent.resourceScope?.workspaceId` → `resourceScope.workspaceId`; `sourceEvent.tenantId` or `sourceEvent.sourceTenantId` or `sourceEvent.targetTenantId` or `sourceEvent.resourceScope?.tenantId` → `resourceScope.tenantId`; `sourceEvent.objectKey` → `resourceScope.objectKey`. - [ ] T022 Implement internal helper `categoryForEventType(eventType)` in `storage-audit-ops.mjs`. Pure function. Accepts a string. Matches by prefix: `'storage.object.'` or `'storage.bucket.'` → `'data_plane'`; `'storage.error.'` → `'error'`; `'storage.event_notification.'` → `'lifecycle'`; `'storage.usage.'` → `'administrative'`; `'storage.import_export.'` → `'data_plane'`; anything else → `'data_plane'` (fallback, not an error).

## Phase 8 — Query surface

- [ ] T023 Implement and export `queryStorageAuditTrail(context, params)` in `storage-audit-ops.mjs`. Async function. Execution order: (1) call `assertAuditQueryScope(context, params)` — throws `AUDIT_SCOPE_UNAUTHORIZED` on scope violation; (2) call `normalizeQueryParams(params)` — throws `AUDIT_QUERY_INVALID` on malformed input; (3) emit meta-audit event via `emitStorageAuditEvent(buildStorageMetaAuditEvent(context, normalizedParams), context)`; (4) call `await context.queryAuditRecords(normalizedParams)` — receives `{ items, cursor, total }`; (5) for non-superadmin callers, filter `items` to drop any event where `event.resourceScope.tenantId !== context.tenantId` (cross-tenant leak guard, FR-020); (6) return `{ items: filteredItems, cursor, total, appliedFilters: normalizedParams }`. Per FR-014–FR-018, FR-026.

When `params.limit > 500`, throw `AUDIT_QUERY_INVALID` with a message indicating the maximum. When `params.limit < 1`, coerce to `1` (do not throw).
- [ ] T025 Ensure `queryStorageAuditTrail` enforces `sortOrder` defaulting to `'desc'` when absent. Accepted values: `'asc'`, `'desc'`. Throw `AUDIT_QUERY_INVALID` for any other value. Per FR-016.
